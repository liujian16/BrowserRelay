# 六、浏览器侧执行器（核心）

## 6.1 WS 接收

```js
ws.onmessage = async (event) => {
  const msg = JSON.parse(event.data)

  if (msg.type === "proxy_request") {
    handleProxyRequest(msg)
  }
}
```

---

## 6.2 执行请求

```js
async function handleProxyRequest(req) {
  try {
    const response = await fetch(req.url, {
      method: req.method,
      headers: req.headers,
      body: decodeBase64(req.body),
      credentials: "include"
    })

    const body = await response.arrayBuffer()

    ws.send(JSON.stringify({
      type: "proxy_response",
      requestId: req.requestId,
      status: response.status,
      headers: Object.fromEntries(response.headers.entries()),
      body: encodeBase64(body)
    }))

  } catch (e) {
    ws.send(JSON.stringify({
      type: "proxy_error",
      requestId: req.requestId,
      message: e.message
    }))
  }
}
```

# 二、核心设计变化

## 从“消息模型”升级为“流模型”

原来：

```text
request → 一条JSON → response
```

现在：

```text
request_start

request_end

response_start
response_chunk *
response_end
```

👉 这是一个**多帧协议（framed streaming protocol）**

# 四、Server 端实现（关键）

## 4.1 /proxy 接收流（Node/Go 都类似）

伪代码：

```text
on /proxy request:

  reqId = uuid()
  send WS proxy_request_start

  for chunk in HTTP request body stream:
      send WS proxy_request_chunk

  send WS proxy_request_end

  ↓

  等待 response_start

  将 response 写回 HTTP response（流式）
```

---

## 4.2 HTTP → WS（关键点）

### Node.js


### Go

```go

```

👉 不允许：

* `await req.body()` ❌
* `collect all` ❌

```text

    res.writeHead(status, headers)

on proxy_response_chunk:
    res.write(chunk)

on proxy_response_end:
    res.end()
```

# 五、浏览器端（最难点）

浏览器必须做到两件事：

---

## 5.1 将 WS → ReadableStream

```js
function createStreamFromWS(requestId) {
  return new ReadableStream({
    start(controller) {
      wsHandlers[requestId] = {
        onChunk: (chunk) => controller.enqueue(chunk),
        onEnd: () => controller.close(),
        onError: (e) => controller.error(e)
      }
    }
  })
}
```

---

## 5.2 fetch 使用流作为 body

```js
const stream = createStreamFromWS(req.requestId)

fetch(req.url, {
  method: req.method,
  headers: req.headers,
  body: stream,
  duplex: "half"   // ⚠️ 必须（Chrome要求）
})
```

👉 注意：

* `duplex: "half"` 是关键（否则流式 POST 不工作）
* 这是浏览器里最容易踩坑的点

# 七、背压（Backpressure）问题

这是你这个架构最容易被忽略但最关键的点。

## 问题

如果：

```text
Local Process 写入很快
Browser fetch 消费很慢
```

👉 会爆内存

---

## 解决方案（必须实现）

### 方案：ACK / Window 控制

#### 浏览器发 ACK：

```json
{
  "type": "proxy_ack",
  "requestId": "req-1",
  "window": 65536
}
```

#### Server 控制发送速率：

```text
if (inflight_bytes > window):
    pause HTTP read
```

---

## Node 实现

```js
req.pause()
req.resume()
```

---

## Go 实现

用：

```go
io.LimitedReader + channel 控制
```
